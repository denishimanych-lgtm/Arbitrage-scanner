# UI/UX Требования v2: Telegram-бот арбитражного сканера

**Дата:** 2026-01-05
**Версия:** 2.0
**Статус:** Production

---

## Философия UI v2

**Принципы:**
1. **Уведомления — главное.** Пользователь получает алерты автоматически, не нужно открывать "списки спредов"
2. **Минимум кликов.** Убрана лишняя вложенность меню
3. **Статус на виду.** Здоровье системы видно сразу на главном экране
4. **Настройки там где нужны.** Безопасность и уведомления — главные действия

---

При старте бота (`/start`) пользователь видит:

- Добавлен live-статус системы
- Убран "Мониторинг" — слился с "Уведомления"

## Убрано из v1

1. **"📊 Арбитраж" кнопка** — не нужна, уведомления приходят автоматически
2. **"Все спреды"** — избыточно при наличии уведомлений
3. **"По типам"** — сложно, типы выбираются в настройках уведомлений
4. **Отдельные модули DEX/Spot/Futures/PerpDEX** — объединены
5. **"Общие настройки" на главной** — переехало в футер
6. **"Мониторинг"** — слился с Уведомлениями
7. **"Отслеживаемые"** — убрано (подписки на отдельные монеты мало используются)
